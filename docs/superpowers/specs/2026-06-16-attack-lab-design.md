# Attack Lab — Design Spec

**Date:** 2026-06-16
**Covers:** Design spec §7 (Attack Lab Scenarios) and §9 end-to-end validation

## 1. Overview

Eight independent attack simulation scripts — three bash (Linux VM) and five PowerShell (Windows VM) — each targeting one of the eight existing detection rules (`RULE-001` through `RULE-008`). A `validate.py` helper polls a new `/api/alerts` JSON endpoint on the SIEM to confirm each rule fires, writing results to `attack-lab/COVERAGE.md`. A summary of that table is copied into the root `README.md` during §10 README polish.

All scripts are written now for future execution on VMs that are not yet set up.

## 2. Scope

**In scope:**
- `attack-lab/` directory with 8 scenario folders, each containing one script
- `attack-lab/validate.py` — polls SIEM, writes COVERAGE.md
- `attack-lab/COVERAGE.md` — auto-generated coverage table
- `GET /api/alerts` JSON endpoint added to the Flask app (`app/dashboard/routes.py`)

**Out of scope:**
- §10 README polish (coverage table excerpt, architecture diagram, screenshots) — separate cycle
- SSH/remote execution of attack scripts from `validate.py` — scripts are run manually on target VMs
- Dashboard auth, pagination, or additional API endpoints

## 4. Attack Scripts

Each script is self-contained. It prints what it does to stdout and cleans up after itself (removes test users, temp files, scheduled tasks, dump files). Scripts require the relevant forwarder to be running and pointed at the SIEM.

### Prerequisites

**Linux VM:** `linux_forwarder.py` running, `SIEM_URL` set, user has `sudo` access.

**Windows VM:** Sysmon installed and running, `windows_forwarder.py` running, `SIEM_URL` set. Script 07 additionally requires `procdump.exe` (Sysinternals, free download) in the same folder as `run.ps1` and must be run as Administrator.

### 01 — SSH Brute Force (`run.sh`)

Loops `ssh` with a non-existent user 6 times (1s apart) against `localhost`, generating 6 failed SSH entries in `/var/log/auth.log`. Triggers RULE-001's aggregation threshold (≥5 `auth_failure` events from the same `src_ip` within 60s).

```bash
TARGET="nonexistent@localhost"
for i in $(seq 1 6); do
    ssh -o BatchMode=yes -o ConnectTimeout=2 -o StrictHostKeyChecking=no "$TARGET" 2>/dev/null || true
    sleep 1
done
echo "Done: 6 failed SSH attempts sent"
```

**Triggers:** RULE-001 (T1110 — Brute Force)

### 02 — Sudo Shadow Edit (`run.sh`)

Runs `sudo grep root /etc/shadow`. The sudo log records `COMMAND=.../grep root /etc/shadow`, which contains "shadow" and matches RULE-002's `regex: "(shadow|visudo)"`. `grep` is used instead of `visudo` because `visudo` opens an interactive editor.

```bash
sudo grep root /etc/shadow > /dev/null
echo "Done: sudo access to /etc/shadow logged"
```

**Triggers:** RULE-002 (T1548.003 — Sudo Caching)

### 03 — New Local User (`run.sh`)

Creates then immediately deletes a test user. The sudo log records `COMMAND=.../useradd attack-lab-user`, which contains "useradd" and matches RULE-003.

```bash
sudo useradd attack-lab-user
sudo userdel -r attack-lab-user 2>/dev/null || sudo userdel attack-lab-user
echo "Done: useradd + userdel executed"
```

**Triggers:** RULE-003 (T1136.001 — Create Account)

### 04 — Encoded PowerShell (`run.ps1`)

Base64-encodes a benign command and runs it with `-enc`. Sysmon Event ID 1 captures `powershell.exe` with `-enc` in the command line, matching RULE-004.

```powershell
$cmd = "Write-Output 'attack-lab'"
$enc = [Convert]::ToBase64String([Text.Encoding]::Unicode.GetBytes($cmd))
powershell.exe -enc $enc
Write-Host "Done: powershell -enc executed"
```

**Triggers:** RULE-004 (T1059.001 — PowerShell)

### 05 — Office Spawns Shell (`run.ps1`)

**Simulated.** Copies `cmd.exe` to `$env:TEMP\winword.exe`, then launches that copy with `cmd.exe` as a child process. Sysmon Event ID 1 captures `cmd.exe` with `ParentImage` ending in `winword.exe`. The forwarder's `_basename()` extracts `winword.exe`, matching RULE-005's `parent_process in ["winword.exe", "excel.exe"]`. Cleans up the copy afterward.

```powershell
$fake = "$env:TEMP\winword.exe"
Copy-Item "$env:SystemRoot\System32\cmd.exe" $fake
& $fake /c "echo attack-lab"
Remove-Item $fake -Force
Write-Host "Done: cmd.exe spawned from simulated winword.exe"
```

**Triggers:** RULE-005 (T1059 — Execution via Office macro simulation)

### 06 — Scheduled Task (`run.ps1`)

Creates then immediately deletes a scheduled task via `schtasks.exe`. Sysmon Event ID 1 captures `schtasks.exe` with `/create` in the command line, matching RULE-006.

```powershell
schtasks /create /tn "AttackLabTask" /tr "cmd.exe" /sc once /st 00:00 /f | Out-Null
schtasks /delete /tn "AttackLabTask" /f | Out-Null
Write-Host "Done: scheduled task created and deleted"
```

**Triggers:** RULE-006 (T1053.005 — Scheduled Task)

### 07 — LSASS Memory Dump (`run.ps1`)

**Requires `procdump.exe` in the same folder and Administrator rights.** Runs `procdump.exe -accepteula -ma lsass.exe lsass.dmp`, generating a Sysmon Event ID 1 with "lsass" in the command line, matching RULE-007. Deletes the dump file immediately after.

```powershell
$here = Split-Path -Parent $MyInvocation.MyCommand.Path
& "$here\procdump.exe" -accepteula -ma lsass.exe lsass.dmp | Out-Null
Remove-Item lsass.dmp -Force -ErrorAction SilentlyContinue
Write-Host "Done: procdump against lsass executed"
```

**Triggers:** RULE-007 (T1003.001 — LSASS Memory)

### 08 — C2 Port Connection (`run.ps1`)

Attempts a TCP connection to port 4444 on localhost (connection will be refused — the attempt itself is enough). Sysmon Event ID 3 captures a network connection event from `powershell.exe` to port 4444, matching RULE-008's `dest_port in [4444, 4445]`.

```powershell
$tcp = New-Object System.Net.Sockets.TcpClient
try { $tcp.Connect("127.0.0.1", 4444) } catch {}
$tcp.Close()
Write-Host "Done: outbound TCP connection to port 4444 attempted"
```

**Triggers:** RULE-008 (T1071 — C2 Port)

## 5. `/api/alerts` Endpoint

New route added to `app/dashboard/routes.py`:

```
GET /api/alerts
```

**Query parameters (both optional):**
- `rule_id` — filter by exact rule ID (e.g., `RULE-001`)
- `since` — ISO 8601 timestamp; returns only alerts with `created_at >= since`

**Response:** JSON array of alert objects:

```json
[
  {
    "id": 42,
    "rule_id": "RULE-001",
    "title": "SSH Brute Force",
    "severity": "medium",
    "created_at": "2026-06-16T10:01:33",
    "host": "linux-vm"
  }
]
```

Empty array `[]` if no alerts match. No auth, no pagination.

## 6. `validate.py`

Run from any machine with Python 3 and network access to the SIEM. Uses only stdlib (`urllib`, `datetime`, `argparse`) — no pip install required.

```
python attack-lab/validate.py --siem http://<siem-ip>:5000
python attack-lab/validate.py --siem http://<siem-ip>:5000 --scenario 01
```

**Per-scenario flow:**

1. Print scenario header (number, name, ATT&CK technique)
2. Record `started_at = datetime.utcnow().isoformat()`
3. Prompt: "Run `attack-lab/NN-<name>/run.sh` (or `.ps1`) on the <VM> VM, then press Enter"
4. Poll `GET /api/alerts?rule_id=RULE-NNN&since=<started_at>` every 5s, up to 60s timeout
5. Alert found → print `✅ RULE-NNN fired (alert id=<id>)`; timeout → print `❌ RULE-NNN did not fire within 60s`
6. After all scenarios (or `--scenario` subset), rewrite `attack-lab/COVERAGE.md` with updated results

**Scenario manifest** (embedded in `validate.py`):

```python

    {"num": "01", "name": "SSH Brute Force",       "vm": "Linux",   "rule": "RULE-001", "technique": "T1110"},
    {"num": "02", "name": "Sudo Shadow Edit",       "vm": "Linux",   "rule": "RULE-002", "technique": "T1548.003"},
    {"num": "03", "name": "New Local User",         "vm": "Linux",   "rule": "RULE-003", "technique": "T1136.001"},
    {"num": "04", "name": "Encoded PowerShell",     "vm": "Windows", "rule": "RULE-004", "technique": "T1059.001"},
    {"num": "05", "name": "Office Spawns Shell",    "vm": "Windows", "rule": "RULE-005", "technique": "T1059"},
    {"num": "06", "name": "Scheduled Task",         "vm": "Windows", "rule": "RULE-006", "technique": "T1053.005"},
```
